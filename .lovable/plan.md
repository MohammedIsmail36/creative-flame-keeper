## خطة التنفيذ: مؤشر الفترة الجارية + حماية حسابات النظام

### المهمة الأولى: إضافة تنبيه "الفترة الجارية" في ميزان المراجعة وقائمة الدخل

عند تفعيل إقفال السنة المالية ووجود قيد إقفال سابق (`lastClosingDate` موجود)، يظهر شريط تنبيه (Alert) أسفل KPI cards وأعلى الجدول:

- **النص**: "البيانات المعروضة للفترة الجارية فقط بعد آخر إقفال بتاريخ {lastClosingDate}"
- **التصميم**: شريط info خفيف (أيقونة `Info` + خلفية زرقاء فاتحة) يتماشى مع تصميم الداشبورد
- **الشرط**: يظهر فقط عندما `settings.enable_fiscal_year_closing && lastClosingDate && !dateFrom`
- **الملفات**: `TrialBalance.tsx` و `IncomeStatement.tsx`

### المهمة الثانية: حماية حسابات النظام من الحذف

رأيي كخبير نظم: **نعم، هذا ضروري جداً**. الحسابات التي يعتمد عليها النظام في إنشاء القيود الآلية (مثل الصندوق، البنك، العملاء، الموردين، المخزون، الإيرادات، تكلفة البضاعة، الأرباح المحتجزة) يجب أن تكون محمية من الحذف والتعطيل لأن حذفها سيكسر كل العمليات التشغيلية.

**الأسلوب**: إضافة عمود `is_system` (boolean) لجدول الحسابات بدلاً من hardcode في الكود، مما يتيح مرونة مستقبلية.

#### التفاصيل:

1. **Migration SQL**: إضافة عمود `is_system boolean NOT NULL DEFAULT false` لجدول `accounts`، ثم تحديث الحسابات المعروفة:

```sql
ALTER TABLE accounts ADD COLUMN is_system boolean NOT NULL DEFAULT false;

UPDATE accounts SET is_system = true 
WHERE code IN ('1101', '1102', '1103', '1104', '2101', '3101', '3102', '4101', '5101', '5201', '4201');
```

2. **تحديث `Accounts.tsx**`:
  - في `handleDelete`: إذا كان `account.is_system === true` → عرض رسالة "هذا حساب نظام أساسي ولا يمكن حذفه"
  - إخفاء زر الحذف للحسابات النظامية أو إظهاره معطّلاً
  - إضافة Badge "نظام" بجانب اسم الحساب النظامي
  - منع تعطيل (`is_active = false`) حسابات النظام أيضاً
3. **تحديث واجهة Account**: إضافة `is_system` للـ interface
