

## تقرير المخزون — حل مبسَّط مع 4 مستويات تنبيه

### الفلسفة
**شفافية كاملة بدلاً من توزيع تلقائي**: نعرض القيمتين (التشغيلية والمحاسبية) ونرشد المستخدم، مع تصعيد التنبيه حسب حجم الفرق.

---

### 1. حساب القيمتين بشكل مستقل

**القيمة التشغيلية (لكل منتج)**
- `WAC = آخر unit_cost من inventory_movements` (نوع: purchase / opening_balance).
- `stock_value للمنتج = quantity_on_hand × WAC`.
- مجموع التقرير = Σ stock_value.

**القيمة المحاسبية (من GL)**
- استعلام واحد لرصيد حساب `1104` من `journal_entry_lines` للقيود `posted` (مع احترام فلتر إقفال السنة).

---

### 2. شريط التحقق المحاسبي — 4 مستويات

| المستوى | الشرط | اللون | الرسالة |
|---|---|---|---|
| **متطابق** | `diff = 0` | أخضر | ✓ المخزون متطابق محاسبياً |
| **تقريب مقبول** | `0 < |diff| ≤ 1` | أخضر فاتح | ✓ متطابق (فرق تقريب: X ج.م) |
| **فرق ملموس** | `1 < |diff|` و `|diff| ≤ 5% من GL` و `|diff| ≤ 1000` | أصفر | ⚠ فرق محاسبي + إرشاد + زر "دفتر الأستاذ" |
| **فرق جوهري** | `|diff| > 5% من GL` أو `|diff| > 1000` | **أحمر** | 🚨 فرق جوهري — يتطلب تدقيق فوري + "يُرجى إيقاف إقفال الشهر حتى تسوية الفرق مع إدارة المالية" |

**نموذج المستوى الأحمر:**
```text
┌──────────────────────────────────────────────────────────┐
│ 🚨 فرق جوهري — يتطلب تدقيق فوري                          │
│ التقرير: 7,150  •  دفتر الأستاذ: 8,250  •  فرق: 1,100   │
│ نسبة الانحراف: 13.3% من رصيد المخزون                     │
│                                                          │
│ ⛔ يُرجى إيقاف إقفال الشهر حتى تسوية هذا الفرق           │
│    مع إدارة المالية.                                      │
│                                                          │
│ خطوات التدقيق:                                            │
│ 1. راجع كشف حساب 1104     [فتح دفتر الأستاذ] ←زر         │
│ 2. راجع آخر تسويات المخزون [فتح التسويات]   ←زر         │
│ 3. تأكد من ترحيل كل قيود PPV ومرتجعات الشراء              │
└──────────────────────────────────────────────────────────┘
```

---

### 3. KPIs (مصدر موحَّد = GL)

| البطاقة | المصدر | الملاحظة |
|---|---|---|
| قيمة المخزون | **رصيد GL لحساب 1104** | متطابق مع الميزانية و Dashboard |
| قيمة البيع | `Σ qty × selling_price` | كما هي |
| الربح المتوقع | `قيمة البيع − GL` | متسق |
| متوسط WAC للوحدة | `GL / Σ qty_on_hand` | معلوماتي |

---

### 4. جدول المنتجات (تشغيلي تفصيلي)
- يبقى يعرض WAC لكل منتج من حركاته (للقرارات التشغيلية).
- **تذييل الجدول** يعرض ثلاث قيم متجاورة:
  ```text
  مجموع التشغيلي: 8,150  │  دفتر الأستاذ: 8,250  │  الفرق: 100
  ```

---

### 5. ما تم استبعاده (تبسيط مقصود)
- ❌ التوزيع النسبي التلقائي للفروق على المنتجات.
- ❌ حساب `adjusted_wac`.
- ❌ زر "تشغيل تسوية تلقائية".
- ✅ استُبدل بإرشاد واضح + روابط للصفحات المعنية.

### مزايا الحل
- **بساطة**: لا خوارزميات توزيع.
- **شفافية**: المستخدم يرى الرقمين والفرق صراحة.
- **تصعيد ذكي**: 4 مستويات تنبيه تتناسب مع خطورة الفرق.
- **حماية للإقفال**: تنبيه أحمر صريح يمنع إقفال شهر بفروق جوهرية.
- **آمن محاسبياً**: لا يخفي قيود يدوية أو أخطاء PPV.
- **تطبيق سريع**: تعديل ملف واحد رئيسي.
