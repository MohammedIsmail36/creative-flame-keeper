# خطة: إضافة حقلَي "مسمى الباركود" و "سعر الباركود" للمنتج

الهدف: تجهيز المنتج لطباعة ملصقات الباركود على طابعات الرول (XPrinter) بحيث يظهر:
- سطر مختصر أعلى الباركود (مثل: `قميص فورسيزون - 280`)
- سعر مطبوع على الملصق مستقل تماماً عن سعر البيع الافتراضي في النظام

## 1. قاعدة البيانات

إضافة عمودين جديدين لجدول `products`:
- `barcode_label TEXT NULL` — النص المختصر أعلى الباركود
- `barcode_price NUMERIC(12,2) NULL` — السعر المطبوع على الملصق فقط، **لا يؤثر** على `selling_price` ولا على أي منطق فواتير/محاسبة/تقارير

تحديث `public/full-schema.sql` بنفس التغييرات.

## 2. الواجهة (Product Form + View)

`src/pages/ProductForm.tsx`: قسم فرعي جديد "إعدادات ملصق الباركود" يضم:
- حقل `barcode_label` (نصي) + زر "اقتراح تلقائي" ينتج: `<أول كلمة/كلمتين من الاسم> <الماركة> - <رقم الموديل>` (قابل للكتابة فوقه).
- حقل `barcode_price` (رقمي) مع تلميح: "يظهر على ملصق الباركود فقط، لا يغيّر سعر البيع".
- كلا الحقلين اختياريان.

`src/pages/ProductView.tsx`: عرض القيمتين (إن وُجدتا) في بطاقة تفاصيل المنتج.

## 3. الاستيراد / التصدير

`src/pages/Products.tsx` (تصدير Excel/CSV): إضافة عمودين جديدين `barcode_label` / "مسمى الباركود" و `barcode_price` / "سعر الباركود".

`src/pages/ProductImport.tsx`:
- قبول العمودين بالأسماء العربية أو الإنجليزية.
- عند إنشاء منتج جديد: تخزين القيم كما هي.
- عند تحديث منتج موجود: تحديث القيم فقط إذا وردت في الملف (نفس منطق الباركود الحالي).

## 4. طباعة الملصقات (خارج نطاق هذه المهمة)

الحقول جاهزة فقط. شاشة الطباعة تُبنى لاحقاً وتقرأ:
- `barcode_label` (Fallback: `formatProductDisplay(name, brand, model)`).
- `barcode_price` (Fallback: `selling_price`).

## تفاصيل تقنية

- Migration واحدة:
  ```sql
  ALTER TABLE public.products
    ADD COLUMN barcode_label TEXT,
    ADD COLUMN barcode_price NUMERIC(12,2);
  ```
- لا سياسات RLS جديدة (يتبع صلاحيات المنتج).
- لا تغيير على المحاسبة، المخزون، الفواتير، أو أي حساب أسعار/تكاليف.
- `types.ts` يتحدث تلقائياً بعد الميجريشن.

## المخاطر

منخفضة جداً: حقلان اختياريان معزولان عن أي منطق قائم. الاستيراد يبقى متوافقاً مع الملفات القديمة.
