## المشكلة

يوجد **ثلاث معادلات مختلفة** للمخزون المنخفض في النظام:

| الموقع | المعادلة الحالية |
|---|---|
| لوحة التحكم — عدّاد KPI (`Dashboard.tsx:576-582`) | `qty <= min && min > 0` |
| لوحة التحكم — قائمة "مخزون منخفض" (`Dashboard.tsx:793-797`) | `qty <= min && min > 0` |
| قائمة المنتجات — فلتر (`Products.tsx:229, 433`) | `qty > 0 && qty < min` |
| كارت المنتج — شارة "منخفض" (`ProductCard.tsx`) | `qty > 0 && qty < min` |

## القرار المعتمد

**التعريف الموحّد:** `quantity_on_hand < min_stock_level && min_stock_level > 0` (يستثني `qty == min`، ويستثني الأصناف بدون حد أدنى، ويشمل النافذ لكن النافذ يُعرض بشارة مستقلة عند اللزوم).

## التغييرات

**ملف واحد:** `src/pages/Dashboard.tsx`

1. **عدّاد KPI (السطر 576-582):** استبدال `qty <= min` بـ `qty < min` مع الإبقاء على شرط `min > 0`.

2. **قائمة "مخزون منخفض" (السطر 793-797 داخل `fetchLowStock`):** نفس التعديل — `qty < min && min > 0`.

بقية المواقع (Products.tsx والكارت) صحيحة ولا تحتاج تعديل.

## التحقق

- منتج بـ `qty=5, min=5` → لن يظهر في أي مكان كـ"منخفض" (كان يظهر في لوحة التحكم فقط).
- منتج بـ `qty=3, min=5` → يظهر في الاثنين.
- منتج بـ `qty=0, min=5` → يظهر كـ"نفذ" في الكارت، ويظل ضمن قائمة المخزون المنخفض في لوحة التحكم (لأن `0 < 5`).
