

# إضافة خيار تفعيل/تعطيل التحقق من فترة الاسترجاع

## المشكلة
عند اعتماد مرتجع مبيعات، النظام يتحقق من أن الكمية المرتجعة لا تتجاوز الكمية المباعة خلال آخر N يوم. هذا يمنع إرجاع أصناف بيعت قبل بدء استخدام النظام لأنها غير مسجلة في قاعدة البيانات.

## الحل
إضافة مفتاح تفعيل/تعطيل (Switch) في الإعدادات بجانب حقل "مدة الاسترجاع"، بحيث يمكن تعطيل هذا التحقق بالكامل عند الحاجة.

## التغييرات المطلوبة

### 1. Migration — إضافة عمود `enable_return_days_limit`
إضافة عمود `boolean` جديد في جدول `company_settings`:
```sql
ALTER TABLE company_settings 
ADD COLUMN enable_return_days_limit boolean NOT NULL DEFAULT true;
```

### 2. تحديث `SettingsContext.tsx`
إضافة الحقل الجديد `enable_return_days_limit: boolean` لنوع `CompanySettings`.

### 3. تحديث صفحة الإعدادات `SettingsPage.tsx`
إضافة Switch بجانب حقل مدة الاسترجاع:
- عند التعطيل: يصبح حقل عدد الأيام معطلاً (disabled) ويظهر تنبيه توضيحي
- عند التفعيل: يعمل كالمعتاد

### 4. تحديث `SalesReturnForm.tsx`
تعديل دالة `postReturn` بحيث يتم تخطي كتلة التحقق من الكمية المباعة بالكامل إذا كان `enable_return_days_limit === false`:
```typescript
if (settings?.enable_return_days_limit !== false) {
  // ... التحقق الحالي من الكمية المباعة خلال الفترة
}
```

## النتيجة
- عند **تفعيل** الخيار: النظام يعمل كالسابق ويمنع الإرجاع إذا تجاوزت الكمية
- عند **تعطيل** الخيار: يمكن إرجاع أي صنف بأي كمية بدون تحقق من فترة البيع، مع الاستمرار بالتحقق من توفر المخزون الفعلي
