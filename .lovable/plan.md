

# خطة إعادة بناء تقرير المبيعات (النسخة المحدّثة)

تم دمج جميع ملاحظات المراجعة وتصحيح الثغرات المذكورة.

| الملف | التغيير |
|-------|---------|

---

## 1. الاستعلامات (3 استعلامات)

| # | الجدول | البيانات | ملاحظات |
|---|--------|----------|---------|
| 1 | `sales_invoices` + `customers(name)` + `sales_invoice_items(quantity, total, product_id, product:products(name))` | الفواتير والبنود | فلتر بالتاريخ، بدون `.limit()` لأن DataTable يتعامل مع Pagination محلياً |
| 2 | `sales_returns` (status='posted') + `customers(name)` + `sales_return_items(quantity, total, product_id)` | المرتجعات | فلتر بنفس نطاق التاريخ |
| 3 | `inventory_movements` (movement_type IN sale, sale_return) | التكلفة الفعلية COGS | فلتر بنفس نطاق التاريخ |

الاستعلامات الثلاثة تعمل بـ `useQuery` بشكل متوازي. التصفية والتجميع عبر `useMemo` — لا re-fetch عند تبديل الوضع.

---

## 2. الفلاتر

- **من تاريخ / إلى تاريخ** (موجود — DatePickerInput)
- **فلتر الحالة**: الكل، مُرحّل، مسودة، ملغي (Select جديد)
- **فلتر العميل**: LookupCombobox قابل للبحث (جديد)
- **تجميع حسب**: الفاتورة، العميل، المنتج، يومي (محدّث)

تصميم الفلاتر يتبع نفس نمط ProductAnalytics المحسّن (فواصل بصرية + labels).

---

## 3. بطاقات KPI (6 بطاقات)

| البطاقة | الحساب |
|---------|--------|
| عدد الفواتير | count الفواتير المُرحّلة |
| إجمالي المبيعات | sum(total) للمُرحّلة |
| المرتجعات | sum(total) من sales_returns المُرحّلة |
| صافي المبيعات | إجمالي - مرتجعات |
| إجمالي الربح | صافي المبيعات - COGS |
| نسبة التحصيل | **(المدفوع / صافي المبيعات) × 100%** — تصحيح: القسمة على الصافي وليس الإجمالي |

## 4. أوضاع التجميع (4 أوضاع)

### 4.1 بالفاتورة
| العمود | ملاحظة |
|--------|--------|
| رقم الفاتورة (رابط للفاتورة) | navigate to `/sales/${id}/edit` |
| التاريخ | |
| العميل | |
| الحالة | Badge ملون |
| الإجمالي | |
| المدفوع | |
| المتبقي | أحمر إذا > 0 |
| متأخر | Badge أحمر إذا due_date < today والمتبقي > 0 |

تلوين الصفوف: مسودة = رمادي، ملغي = خط مشطوب (`line-through opacity-50`).

### 4.2 بالعميل
الأعمدة: العميل، عدد الفواتير، الإجمالي، المرتجعات، الصافي، المدفوع، المتبقي، نسبة التحصيل.

### 4.3 بالمنتج (مبسّط — لا يكرر ProductAnalytics)
الأعمدة: المنتج، الكمية المباعة، المرتجع، صافي الكمية، الإيرادات.
**بدون تكلفة وربح** — مع ملاحظة نصية "للتفاصيل راجع تقرير تحليل المنتجات".

### 4.4 يومي/شهري (جديد)
- **Toggle واضح**: زر يومي / شهري (ToggleGroup)
- يومي: تجميع بـ `invoice_date` مباشرة
- شهري: تجميع بـ `format(date, 'yyyy-MM')` + عرض اسم الشهر بالعربية
- الأعمدة: التاريخ/الشهر، عدد الفواتير، الإجمالي، المرتجعات، الصافي
- **BarChart**: أعمدة المبيعات + أعمدة المرتجعات بلون مختلف

## 6. الجداول — DataTable موحد

جميع الأوضاع تستخدم `DataTable` مع:
- `pageSize={20}`, `showSearch`, `showPagination`
- ColumnDef مع `useMemo` لكل وضع

- أعمدة التكلفة والربح **تختفي** عند اختيار حالة غير "مُرحّل" (تصحيح التعارض)
