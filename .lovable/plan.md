# خطة: حاسبة عمولة البائع (تحت التقارير)

## الفكرة العامة
صفحة تقرير جديدة تحت **/reports/commission** تحسب عمولة البائع تلقائياً بناءً على:
- **المبيعات الشهرية الفعلية** (مسحوبة من قاعدة البيانات — نفس منطق تقرير المبيعات)
- **هامش الربح الفعلي** (محسوب من صافي المبيعات − COGS بطريقة WAC)
- **الهدف الشهري** (من `company_settings.monthly_sales_target`)
- **شرائح العمولة** التي يدخلها المستخدم

بائع واحد فقط للشركة — لا حاجة لجدول بائعين أو تحديد بائع.

## المسار والوصول
- المسار: `/reports/commission`
- ضمن قسم **التقارير** في `AppSidebar` (مع باقي تقارير المبيعات)
- الصلاحيات: `admin`, `accountant`, `sales`
- إضافة label عربي في `route-labels.ts`: `commission: "حاسبة العمولة"`

## الملفات
1. **جديد**: `src/pages/reports/CommissionCalculatorPage.tsx`
2. **تعديل**: `src/App.tsx` — lazy route تحت `/reports/commission`
3. **تعديل**: `src/components/layout/AppSidebar.tsx` — رابط تحت قسم التقارير
4. **تعديل**: `src/lib/route-labels.ts` — إضافة المفتاح

## مصادر البيانات (نفس منطق SalesReport)
الاستعلامات بالتوازي عبر React Query لشهر مختار (افتراضياً الشهر الحالي):

1. **صافي المبيعات (Net Sales)** =
   `sum(sales_invoice_items.net_total)` للفواتير `status='posted'` خلال الشهر
   **ناقص** `sum(sales_return_items.net_total)` للمرتجعات `status='posted'` خلال الشهر

2. **تكلفة المبيعات (COGS)** =
   مجموع `inventory_movements.total_cost` بقيمة سالبة (`movement_type='sale'`) خلال الشهر
   **ناقص** قيم `movement_type='sale_return'` (تخفّض COGS)

3. **هامش الربح %** = `((NetSales − COGS) / NetSales) × 100`

4. **الهدف الشهري** من `company_settings.monthly_sales_target` (مسحوب عبر `useSettings()` الموجود)

> ملاحظة: هذه المعادلات تطابق سياسة المشروع الموثقة:
> Profit = Net Sales − COGS (WAC) — كما في تقرير المبيعات والـ P&L.

## واجهة المستخدم

### Header
- `PageHeader` بعنوان "حاسبة عمولة البائع" + أيقونة `Calculator` (lucide)
- وصف: "احسب عمولة البائع بناءً على نسبة تحقيق الهدف وهامش الربح"

### كرت "الفترة"
- **شهر/سنة** عبر `Select` (آخر 12 شهر افتراضياً) — يتحكم في الاستعلام
- زر "تحديث" يعيد جلب البيانات

### كرت "البيانات الفعلية (تلقائي)" — للقراءة فقط
شبكة 4 إحصائيات:
- الهدف الشهري (من الإعدادات)
- صافي المبيعات (محسوب)
- هامش الربح الفعلي % (محسوب)
- نسبة الإنجاز %

ملاحظة صغيرة بجانب كل قيمة: "محسوب تلقائياً من فواتير الشهر" مع أيقونة Info.
خيار "تعديل يدوي" (toggle) يفتح حقول قابلة للكتابة لتجاوز القيم في حال احتاج المستخدم سيناريو افتراضي.

### كرت "إعدادات الحاسبة"
- **الحد الأدنى لهامش الربح %** (افتراضي 25%)
- **شرائح العمولة** (3 بطاقات شبكة 3 أعمدة):
  - 100% – 119%  → نسبة افتراضية 3%
  - 120% – 139%  → نسبة افتراضية 4%
  - 140%+         → نسبة افتراضية 5%
  - الشريحة المطبّقة تتميّز بـ `border-primary bg-primary/10`
- يتم حفظ الإعدادات (الشرائح + الحد الأدنى) في `localStorage` لتذكّرها بين الجلسات

### كرت "النتيجة"
- `Progress` لنسبة الإنجاز (أخضر/أصفر/أحمر حسب النسبة)
- 3 إحصائيات: نسبة الإنجاز، الشريحة المطبّقة، العمولة الإجمالية
- صندوق المعادلة: `(المبيعات − الهدف) × نسبة الشريحة = العمولة`
- تنبيهات `Alert`:
  - **destructive**: "لم يتم بلوغ الهدف — لا تُصرف عمولة"
  - **warning**: "هامش الربح أقل من الحد الأدنى — تم إلغاء العمولة"
- جدول `Table` للتفصيل: إجمالي المبيعات / الهدف / الفرق الخاضع / النسبة / العمولة

## المنطق
- العمولة = `(NetSales − Target) × tierRate / 100`
- لا تُصرف إذا: `NetSales < Target` **أو** `Margin < MinMargin` **أو** `NetSales = 0`
- اختيار الشريحة بناءً على `achievement%`:
  - ≥ 140 → الشريحة 3
  - 120–139 → الشريحة 2
  - 100–119 → الشريحة 1
