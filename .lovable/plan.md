

# خطة تحسين المشتريات وإصلاح تهيئة النظام

---

## المشكلة الأساسية: النظام لا يعمل بعد التهيئة

عند تشغيل "تهيئة النظام" (seed-system)، يتم إنشاء حساب المدير في نظام المصادقة لكن **لا يتم إسناد دور "admin" في جدول `user_roles`**. بما أن جميع الجداول محمية بسياسات RLS تتطلب دوراً، فلا يمكن للمستخدم رؤية أو إدخال أي بيانات.

---

## الملفات المتأثرة

| الملف | التغيير |
|-------|---------|
| `supabase/functions/seed-system/index.ts` | إضافة إسناد دور admin + إنشاء profile |
| `src/pages/Dashboard.tsx` | إصلاح خطأ ref في EmptyState |
| `src/pages/reports/PurchasesReport.tsx` | إعادة بناء شاملة بنفس نمط SalesReport |
| `src/pages/reports/PurchasesReportPage.tsx` | تحديث طفيف إن لزم |

---

## 1. إصلاح seed-system (حرج)

بعد إنشاء المستخدم الجديد، إضافة:
- **إدراج في `user_roles`**: `{ user_id, role: 'admin' }`
- **إدراج في `profiles`**: `{ id: user_id, full_name }` (إن وُجد جدول profiles)
- التأكد من عدم التكرار إذا كان الدور موجوداً مسبقاً

---

## 2. إصلاح Dashboard — خطأ EmptyState ref

المشكلة: `EmptyState` مكون دالة يُمرر له `ref` ضمنياً عبر `CardContent`. الحل: تحويله إلى `React.forwardRef` أو إزالة السياق الذي يمرر الـ ref.

---

## 3. إعادة بناء تقرير المشتريات

تطبيق نفس التحسينات التي تمت على تقرير المبيعات:

### 3.1 الاستعلامات (3 استعلامات متوازية)
| # | الجدول | البيانات |
|---|--------|----------|
| 1 | `purchase_invoices` + `suppliers(name)` + `purchase_invoice_items(...)` | الفواتير والبنود |
| 2 | `purchase_returns` (status='posted') + `suppliers(name)` | المرتجعات |
| 3 | `inventory_movements` (purchase, purchase_return) | تكلفة المخزون |

### 3.2 الفلاتر (بنفس نمط SalesReport المحسّن)
- من/إلى تاريخ
- فلتر الحالة: الكل، مُرحّل، مسودة، ملغي
- فلتر المورد: LookupCombobox
- تجميع حسب: الفاتورة، المورد، المنتج، زمني (يومي/شهري)

### 3.3 بطاقات KPI (6 بطاقات — صفين)
| البطاقة | الحساب |
|---------|--------|
| عدد الفواتير | count المُرحّلة |
| إجمالي المشتريات | sum(total) |
| المرتجعات | sum(total) من purchase_returns |
| صافي المشتريات | إجمالي - مرتجعات |
| المدفوع | sum(paid_amount) |
| نسبة السداد | (المدفوع / صافي المشتريات) x 100% |

تصميم البطاقات: خلفيات متدرجة + أيقونات ملونة (نفس نمط SalesReport).

### 3.4 أوضاع التجميع (4 أوضاع)

**بالفاتورة**: رقم الفاتورة (رابط + prefix)، التاريخ، المورد، الحالة (Badge)، الإجمالي، المدفوع، المتبقي، متأخر.
- تلوين: مسودة رمادي، ملغي مشطوب.

**بالمورد**: المورد، عدد الفواتير، الإجمالي، المرتجعات، الصافي، المدفوع، المتبقي، نسبة السداد.

**بالمنتج**: المنتج، الكمية المشتراة، المرتجع، صافي الكمية، التكلفة.

**زمني**: Toggle يومي/شهري + BarChart مزدوج (مشتريات + مرتجعات).

### 3.5 الرسوم البيانية
- زمني: BarChart مزدوج
- بالمورد: BarChart أفقي — أكبر 10 موردين
- بالمنتج: BarChart أفقي — أكثر 10 منتجات

### 3.6 DataTable موحد
- `pageSize={20}`, pagination, search
- Footer بإجماليات
- أرقام بتنسيق `en-US`

### 3.7 التصدير
- ExportMenu (CSV + Excel + PDF)
