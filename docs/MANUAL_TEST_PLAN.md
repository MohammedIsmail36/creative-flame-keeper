# خطة الاختبار اليدوي الشاملة - من منظور المستخدم النهائي

## الهدف
اختبار النظام المحاسبي خطوة بخطوة بدءاً من قاعدة بيانات فارغة، مع أرقام محددة مسبقاً وأرقام متوقعة للتحقق منها في كل شاشة وتقرير.

- **نسبة الضريبة**: 15% (يتم ضبطها في الإعدادات)
- **العملة**: ريال سعودي (ر.س)
- **بداية السنة المالية**: 01/01 من السنة الحالية

## المرحلة 3: فاتورة شراء ثانية (بسعر مختلف)

### الهدف
اختبار حساب **المتوسط المرجح للتكلفة (WAC)** عند الشراء بسعر مختلف.

### الخطوات
**الصفحة**: المشتريات → إنشاء فاتورة شراء جديدة

| الحقل | القيمة |
|-------|--------|
| المورد | مورد الاختبار (S001) |
| **بند 1**: المنتج | منتج اختبار أ (P001) |
| الكمية | 100 |
| سعر الوحدة | **60.00** |
| الخصم | 0 |

### الأرقام المتوقعة على شاشة الفاتورة

| البند | القيمة |
|-------|--------|
| الإجمالي الفرعي | **6,000.00** |
| الضريبة (15%) | **900.00** |
| الإجمالي النهائي | **6,900.00** |

- [ ] **تحقق**: أرقام الفاتورة صحيحة

### بعد الترحيل

#### حساب المتوسط المرجح (WAC)
```
WAC = (التكلفة القديمة + التكلفة الجديدة) / إجمالي الوحدات
WAC = (100 × 50 + 100 × 60) / (100 + 100)
WAC = (5,000 + 6,000) / 200
WAC = 11,000 / 200
WAC = 55.00
```

#### أ. المنتج
| البند | القيمة المتوقعة |
|-------|----------------|
| الكمية المتاحة | **200** |
| متوسط سعر الشراء (WAC) | **55.00** |

- [ ] **تحقق**: الكمية = 200، WAC = 55.00

#### ب. رصيد المورد
| البند | القيمة المتوقعة |
|-------|----------------|
| الرصيد | 5,750 + 6,900 = **12,650.00** |

- [ ] **تحقق**: رصيد المورد = 12,650.00

#### ج. حركات المخزون (يجب أن تكون حركتان)
| # | النوع | الكمية | تكلفة الوحدة |
|---|-------|--------|-------------|
| 1 | purchase | 100 | 50.00 |
| 2 | purchase | 100 | 60.00 |

- [ ] **تحقق**: حركتا شراء بالأسعار الصحيحة

#### د. لوحة التحكم
| المؤشر | القيمة المتوقعة |
|--------|----------------|
| إجمالي المشتريات | **11,000.00** أو **12,650.00** (حسب ما إذا كانت تشمل الضريبة) |
| صافي الربح | **0.00** (لا توجد مبيعات بعد) |

- [ ] **تحقق**: لا يوجد ربح بعد، المشتريات مسجلة
- [ ] **سجّل**: القيمة الفعلية لإجمالي المشتريات = _______ (هل تشمل الضريبة أم لا؟)

## المرحلة 4: فاتورة بيع

### الهدف
اختبار أن تكلفة البضاعة المباعة (COGS) تُحسب بالمتوسط المرجح وليس بسعر الشراء الثابت.

### الخطوات
**الصفحة**: المبيعات → إنشاء فاتورة بيع جديدة

| الحقل | القيمة |
|-------|--------|
| العميل | عميل الاختبار (C001) |
| **بند 1**: المنتج | منتج اختبار أ (P001) |
| الكمية | 40 |
| سعر الوحدة | 100.00 |
| الخصم | 0 |

### الأرقام المتوقعة على شاشة الفاتورة

| البند | القيمة |
|-------|--------|
| الإجمالي الفرعي | **4,000.00** |
| الضريبة (15%) | **600.00** |
| الإجمالي النهائي | **4,600.00** |

- [ ] **تحقق**: أرقام الفاتورة صحيحة

### حساب COGS المتوقع
```
COGS = الكمية المباعة × المتوسط المرجح
COGS = 40 × 55.00 = 2,200.00
```

> ⚠️ **نقطة حرجة**: يجب أن تكون تكلفة البضاعة المباعة = **2,200.00** وليس 2,000.00 (40×50).
> إذا ظهرت 2,000 فهناك خطأ - النظام يستخدم سعر الشراء الثابت بدلاً من المتوسط المرجح.

### بعد الترحيل

#### أ. المنتج
| البند | القيمة المتوقعة |
|-------|----------------|
| الكمية المتاحة | 200 - 40 = **160** |
| WAC | **55.00** (لم يتغير) |

- [ ] **تحقق**: الكمية = 160

#### ب. رصيد العميل
| البند | القيمة المتوقعة |
|-------|----------------|
| الرصيد | **4,600.00** |

- [ ] **تحقق**: رصيد العميل = 4,600.00

#### ج. حركة المخزون
| النوع | الكمية | تكلفة الوحدة | الإجمالي |
|-------|--------|-------------|---------|
| sale | 40 | **55.00** | **2,200.00** |

- [ ] **تحقق**: تكلفة الوحدة في الحركة = **55.00** (وليس 50.00 أو 100.00)

#### د. القيد المحاسبي
| الحساب | مدين | دائن | الشرح |
|--------|------|------|-------|
| العملاء (1103) | 4,600.00 | - | إجمالي الفاتورة شاملة الضريبة |
| إيرادات المبيعات (4101) | - | 4,000.00 | أو 4,600.00 حسب معالجة الضريبة |
| ت.ب.م (5101) | **2,200.00** | - | COGS بالمتوسط المرجح |
| المخزون (1104) | - | **2,200.00** | خصم من المخزون |
| ضريبة المخرجات (إن وُجد) | - | 600.00 | |

- [ ] **تحقق**: COGS في القيد = **2,200.00**
- [ ] **تحقق**: القيد متوازن

#### هـ. لوحة التحكم
| المؤشر | القيمة المتوقعة |
|--------|----------------|
| إجمالي المبيعات | **4,000.00** (أو 4,600 إذا شاملة الضريبة) |
| صافي الربح | **إيرادات - تكلفة = 4,000 - 2,200 = 1,800.00** |

- [ ] **سجّل**: إجمالي المبيعات الفعلي = _______
- [ ] **سجّل**: صافي الربح الفعلي = _______

## المرحلة 6: مرتجع شراء

### الهدف
اختبار إرجاع بضاعة للمورد وأثره على المخزون ورصيد المورد.

### الخطوات
**الصفحة**: مرتجعات المشتريات → إنشاء مرتجع جديد

| الحقل | القيمة |
|-------|--------|
| المورد | مورد الاختبار (S001) |
| فاتورة الشراء المرجعية | فاتورة الشراء الثانية (المرحلة 3) |
| **بند 1**: المنتج | منتج اختبار أ (P001) |
| الكمية | 20 |
| سعر الوحدة | 60.00 |

### الأرقام المتوقعة على شاشة المرتجع

| البند | القيمة |
|-------|--------|
| الإجمالي الفرعي | **1,200.00** |
| الضريبة (15%) | **180.00** |
| الإجمالي النهائي | **1,380.00** |

### بعد الترحيل

#### أ. المنتج
| البند | القيمة المتوقعة |
|-------|----------------|
| الكمية المتاحة | 170 - 20 = **150** |

> **ملاحظة حول WAC**: بعد مرتجع الشراء، قد يتغير WAC حسب طريقة الحساب المتبعة.
> - إذا خُصم بسعر المرتجع (60): تكلفة المخزون = (170 × 55) - (20 × 60) = 9,350 - 1,200 = 8,150، WAC = 8,150/150 = **54.33**
> - إذا خُصم بالـ WAC (55): تكلفة المخزون = 150 × 55 = 8,250، WAC = **55.00**
>
> **سجّل الفعلي**: WAC بعد المرتجع = _______

- [ ] **تحقق**: الكمية = 150

#### ب. رصيد المورد
| البند | القيمة المتوقعة |
|-------|----------------|
| الرصيد | 12,650 - 1,380 = **11,270.00** |

- [ ] **تحقق**: رصيد المورد = 11,270.00

#### ج. حركة المخزون
| النوع | الكمية | تكلفة الوحدة |
|-------|--------|-------------|
| purchase_return | 20 | 60.00 أو 55.00 |

- [ ] **سجّل**: تكلفة الوحدة الفعلية في الحركة = _______

## المرحلة 8: مصروف تشغيلي

### الهدف
اختبار تسجيل مصروف وأثره على صافي الربح.

### الخطوات المُسبقة
تأكد من وجود **نوع مصروف** مناسب (مثل "إيجار"). إذا لم يوجد:
**الصفحة**: أنواع المصاريف → إضافة نوع جديد

| الحقل | القيمة |
|-------|--------|
| الاسم | إيجار |
| الحساب | حساب مصروفات (كود 5xxx) |

|---|---------|---------------------|
| 1 | شراء 1 | 5,000 |

| 5 | مرتجع شراء | -1,200 |

| 7 | دفعة مورد | 4,000 |
| 8 | مصروف إيجار | 500 |
| 9 | عجز مخزون | ~165 |
| 10 | قيد يدوي | 200 |

### الأرقام المتوقعة (ملخص)

